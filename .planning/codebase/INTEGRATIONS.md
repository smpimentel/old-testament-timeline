# External Integrations

**Analysis Date:** 2026-02-17

## APIs & External Services

None. This is a fully static, client-side SPA with no backend API calls. All data is bundled at build time from local JSON files.

## Data Sources

**Raw JSON data (hand-authored):**
- `src/data/raw/people.json` - People entities (birth/death years, roles, family trees, scripture refs)
- `src/data/raw/events.json` - Historical events (year, category, description)
- `src/data/raw/books.json` - Old Testament books (date ranges, authors, genres)
- `src/data/raw/periods.json` - Historical periods (date ranges, colors)
- `src/data/raw/themes.json` - Theological themes (colors, descriptions)

**Compiled JSON data (build-generated):**
- `src/data/compiled/people.json` - Transformed from raw, swimlanes assigned
- `src/data/compiled/events.json` - Transformed from raw, swimlanes assigned
- `src/data/compiled/books.json` - Transformed from raw, swimlanes assigned
- `src/data/compiled/periods.json` - Deduped by date range, sorted
- `src/data/compiled/themes.json` - Normalized theme data
- `src/data/compiled/relationships.json` - Extracted from raw data cross-references (family trees, related events/books)

**Data pipeline:**
1. Author raw data in `src/data/raw/*.json`
2. Run `npm run build:data` (`scripts/build-data.ts`) to transform raw -> compiled
3. Run `npm run validate` (`scripts/validate-data.ts`) to validate against JSON schemas in `src/data/schemas/`
4. App imports compiled JSON directly at build time via `src/data/timeline-data.ts`

**JSON Schemas (validation-only, no runtime use):**
- `src/data/schemas/raw/*.raw.schema.json` - Schemas for raw data files
- `src/data/schemas/compiled/*.compiled.schema.json` - Schemas for compiled data files
- Custom lightweight validator in `scripts/validate-data.ts` (no external JSON Schema library)

## Data Flow

```
src/data/raw/*.json
    |
    v (scripts/build-data.ts)
src/data/compiled/*.json
    |
    v (static import in src/data/timeline-data.ts)
React components (via timelineData, periods, themes, relationships exports)
```

## External Fonts

**Google Fonts (loaded at runtime via CSS and HTML):**
- Inter (weights: 400, 700) - Timeline labels, UI text. Loaded in `index.html` and `src/styles/fonts.css`
- Inika (weight: 400) - Serif timeline accent font. Loaded in `index.html`
- Cormorant Garamond (weights: 400, 600, 700) - Display/heading font. Loaded in `src/styles/fonts.css`
- Source Sans 3 (weights: 400, 500, 600, 700) - Body text font. Loaded in `src/styles/fonts.css`
- IBM Plex Mono (weights: 400, 500, 600) - Monospace font. Loaded in `src/styles/fonts.css`

**Font loading strategy:** Dual loading via `<link>` in `index.html` (Inter, Inika with `preconnect`) and `@import url()` in `src/styles/fonts.css` (Cormorant Garamond, Source Sans 3, IBM Plex Mono). This is a potential redundancy/concern.

## Deployment

**Platform:** GitHub Pages (static hosting)

**CI/CD:**
- `.github/workflows/ci.yml` - Runs on push/PR to main/master: lint -> build -> test:coverage (sequential jobs, Node 22)
- `.github/workflows/deploy-pages.yml` - Runs on push to main/master: checkout -> build -> upload artifact -> deploy to GitHub Pages via `actions/deploy-pages@v4`

**Build output:** `dist/` directory, uploaded as GitHub Pages artifact

**Environment config:** No environment variables required. No `.env` files present. All configuration is static/hardcoded.
